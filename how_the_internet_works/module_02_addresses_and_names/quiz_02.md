## 🔑 Answer Key

<details>
<summary>🔓 Click to reveal all answers!</summary>

| # | Correct Answer | Explanation |
|---|---------------|-------------|
| 1 | **B) Internet Protocol** | IP stands for Internet Protocol — the rules for addressing and sending data. |
| 2 | **B) `192.168.1.100`** | Valid IPv4: 4 groups of numbers, each between 0-255. 256 is too high, the third has only 3 groups, and the fourth uses words. |
| 3 | **B) We're running out of IPv4 addresses** | IPv4 has ~4.3 billion addresses. With 5+ billion users and multiple devices each, we need IPv6's virtually unlimited addresses! |
| 4 | **C) `192.168.1.50`** | The `192.168.x.x` range is reserved for private networks. The others are public addresses. |
| 5 | **B) Domain Name System** | DNS is the Domain Name System — it translates domain names to IP addresses. |
| 6 | **B) A phone book that translates domain names to IP addresses** | DNS works like a phone book: you look up a name (domain) and get a number (IP address). |
| 7 | **C) Root → TLD → Authoritative** | DNS lookups go from general to specific: Root Server → TLD Server → Authoritative Server. |
| 8 | **C) The subdomain** | `www` is a subdomain of `youtube.com`. It indicates the web server section. |
| 9 | **B) Top-Level Domain** | TLD = Top-Level Domain. Examples: .com, .org, .edu, .net, .gov |
| 10 | **C) `.edu`** | `.edu` is specifically for educational institutions! |
| 11 | **C) The path** | The path (`/inbox`) tells the server which page you want. |
| 12 | **C) Look up the IP address of a domain name** | `nslookup` performs a DNS lookup and shows you the IP address of any domain! |

</details>
