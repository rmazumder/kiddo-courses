## 🧰 Internet Tools Command Cheat Sheet

Keep this handy! These are real commands used by network engineers and cybersecurity professionals!

### 🔍 `nslookup` — DNS Lookup

**What it does:** Translates a domain name into an IP address (DNS lookup)

```bash
# Basic DNS lookup
nslookup google.com

# Reverse lookup (IP → domain)
nslookup 8.8.8.8

# Use a specific DNS server
nslookup google.com 8.8.8.8

# Look up a specific record type
nslookup -type=MX google.com     # Mail server records
nslookup -type=NS google.com     # Name server records
nslookup -type=AAAA google.com   # IPv6 address records
```

**Example Output:**
```
Server:    192.168.1.1
Address:   192.168.1.1#53

Non-authoritative answer:
Name:    google.com
Address: 142.250.190.14
```

---

### 📡 `ping` — Test Connection

**What it does:** Sends test packets to a server and measures round-trip time

```bash
# Mac/Linux: Send 4 pings
ping -c 4 google.com

# Windows: Send 4 pings (automatic)
ping google.com

# Mac/Linux: Send 10 pings
ping -c 10 google.com

# Windows: Send 10 pings
ping -n 10 google.com

# Ping an IP address directly
ping -c 4 8.8.8.8

# Continuous ping (Ctrl+C to stop)
ping google.com
```

**Example Output:**
```
PING google.com (142.250.190.14): 56 data bytes
64 bytes from 142.250.190.14: icmp_seq=0 ttl=118 time=12.3 ms
64 bytes from 142.250.190.14: icmp_seq=1 ttl=118 time=11.8 ms
64 bytes from 142.250.190.14: icmp_seq=2 ttl=118 time=12.1 ms
64 bytes from 142.250.190.14: icmp_seq=3 ttl=118 time=11.9 ms

--- google.com ping statistics ---
4 packets transmitted, 4 packets received, 0.0% packet loss
round-trip min/avg/max/stddev = 11.8/12.0/12.3/0.2 ms
```

**Understanding the output:**
- `time=12.3 ms` — The round-trip time (lower is better!)
- `0.0% packet loss` — All packets arrived (100% = totally disconnected)
- `ttl=118` — Time To Live (how many more routers the packet can pass through)

---

### 🗺️ `traceroute` / `tracert` — Map the Route

**What it does:** Shows every router (hop) your data passes through

```bash
# Mac/Linux:
traceroute google.com

# Windows:
tracert google.com

# Limit to 15 hops
traceroute -m 15 google.com

# Use ICMP instead of UDP (Mac)
traceroute -I google.com
```

**Example Output:**
```
traceroute to google.com (142.250.190.14), 30 hops max
 1  router.local (192.168.1.1)  1.234 ms
 2  10.0.0.1 (10.0.0.1)  8.456 ms
 3  core-router.isp.com (72.14.215.85)  10.123 ms
 4  google-peer.isp.com (72.14.215.86)  11.567 ms
 5  142.250.190.14 (142.250.190.14)  12.345 ms
```

**Understanding the output:**
- Each numbered line is a **hop** (router)
- The time shows how long it took to reach that router
- `* * *` means the router didn't respond (it's hidden/firewalled)

---

### 📥 `curl` — Fetch Web Content

**What it does:** Fetches data from a web server (like a text-based browser!)

```bash
# Fetch HTTP headers only
curl -I https://google.com

# Fetch a full page (will show raw HTML)
curl https://www.example.com

# Fetch and show just the first 20 lines
curl https://www.example.com | head -20

# Follow redirects
curl -L -I http://google.com

# Show detailed connection info
curl -v https://google.com 2>&1 | head -30

# Check your public IP address
curl ifconfig.me

# Download a file
curl -O https://example.com/file.txt

# Send a custom header
curl -H "User-Agent: MyBrowser/1.0" https://example.com
```

**Example Output (headers only):**
```
HTTP/2 200
content-type: text/html; charset=UTF-8
date: Sat, 21 Feb 2026 12:00:00 GMT
server: gws
```

---

### 🔧 `dig` — Advanced DNS (Bonus!)

**What it does:** A more detailed DNS lookup tool (Mac/Linux only)

```bash
# Basic DNS lookup
dig google.com

# Short answer only
dig +short google.com

# Look up a specific record type
dig google.com MX       # Mail records
dig google.com NS       # Name servers
dig google.com AAAA     # IPv6 address

# Trace the full DNS lookup path
dig +trace google.com

# Use a specific DNS server
dig @8.8.8.8 google.com
```

### 🌐 Browser Developer Tools — Quick Reference

| Action | Chrome/Edge | Firefox | Safari |
|--------|------------|---------|--------|
| Open DevTools | `F12` or `Ctrl+Shift+I` / `Cmd+Option+I` | `F12` or `Ctrl+Shift+I` / `Cmd+Option+I` | `Cmd+Option+I` |
| Element selector | `Ctrl+Shift+C` / `Cmd+Shift+C` | `Ctrl+Shift+C` / `Cmd+Shift+C` | Same |
| Console | `Ctrl+Shift+J` / `Cmd+Option+J` | Same as DevTools | Same as DevTools |
| Device mode | `Ctrl+Shift+M` / `Cmd+Shift+M` | `Ctrl+Shift+M` / `Cmd+Shift+M` | Responsive Design Mode |

**Key DevTools Panels:**
- **Elements** — View and edit HTML/CSS
- **Console** — JavaScript console, error messages
- **Network** — See all HTTP requests/responses
- **Sources** — View JavaScript source files
- **Application** — Cookies, local storage, cache

### 🔍 Finding Your IP Address

```bash
# PUBLIC IP (what the Internet sees):
# Mac/Linux:
curl ifconfig.me
# or visit: https://whatismyipaddress.com

# PRIVATE IP (your local network address):
# Mac:
ipconfig getifaddr en0

# Windows:
ipconfig

# Linux:
hostname -I
```

### ⚡ Quick Network Diagnostic Commands

```bash
# "Is my Internet working?"
ping -c 4 8.8.8.8

# "Is DNS working?"
nslookup google.com

# "How fast is my connection to this server?"
ping -c 10 google.com

# "What path does my data take?"
traceroute google.com     # Mac/Linux
tracert google.com        # Windows

# "What does the server say?"
curl -I https://google.com

# "What's my IP?"
curl ifconfig.me
```

## 🔑 Glossary of Key Terms

| Term | Definition |
|------|-----------|
| **ARPANET** | The first version of the Internet, created in 1969 by the US military |
| **Bandwidth** | How much data a connection can carry at once |
| **Browser** | Software that displays web pages (Chrome, Firefox, Safari, Edge) |
| **Cache** | Stored copies of data for faster access next time |
| **Caesar Cipher** | A simple encryption method that shifts letters by a fixed number |
| **Client** | A device or app that sends requests (your browser, phone, etc.) |
| **CSS** | Cascading Style Sheets — makes web pages look good (colors, fonts, layout) |
| **Data Center** | A building full of servers |
| **DNS** | Domain Name System — translates domain names to IP addresses |
| **DOM** | Document Object Model — the browser's tree representation of a page |
| **Domain Name** | The human-readable name for a website (like youtube.com) |
| **Encryption** | Scrambling data so only authorized people can read it |
| **Ethernet** | A wired connection between a device and router |
| **Fiber Optic** | Thin glass cables that carry data as pulses of light |
| **Firewall** | Software/hardware that blocks unauthorized network access |
| **Front-End** | The visual part of a website (what you see and click) |
| **Back-End** | The server side of a website (databases, processing) |
| **HTML** | HyperText Markup Language — the structure of web pages |
| **HTTP** | HyperText Transfer Protocol — rules for web communication |
| **HTTPS** | HTTP Secure — encrypted web communication |
| **IP Address** | A unique number assigned to every Internet device |
| **IPv4** | Original IP system (4 groups of numbers, ~4.3 billion addresses) |
| **IPv6** | New IP system (8 groups of hex, virtually unlimited addresses) |
| **ISP** | Internet Service Provider (Comcast, AT&T, Verizon, etc.) |
| **JavaScript** | Programming language for web interactivity |
| **NAT** | Network Address Translation — translates private IPs to public |
| **Packet** | A small chunk of data sent over the Internet |
| **Phishing** | Fake messages designed to steal personal information |
| **Ping** | A command that tests connectivity and measures response time |
| **Protocol** | A set of rules for communication |
| **Rendering** | The process of turning code into a visual web page |
| **Router** | A device that directs packets to their destination |
| **Server** | A computer that responds to requests from clients |
| **SSL/TLS** | Encryption technology used by HTTPS |
| **Status Code** | A 3-digit number indicating the result of an HTTP request |
| **TCP/IP** | The fundamental communication protocols of the Internet |
| **TLD** | Top-Level Domain (.com, .org, .edu, .net, etc.) |
| **Traceroute** | A command that shows every router a packet passes through |
| **URL** | Uniform Resource Locator — the full address of a web page |
| **WiFi** | Wireless technology connecting devices to a network |
| **World Wide Web** | The collection of websites accessible via the Internet |
| **2FA** | Two-Factor Authentication — double verification for login |
