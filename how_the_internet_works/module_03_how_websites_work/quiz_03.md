## 🔑 Answer Key

<details>
<summary>🔓 Click to reveal all answers!</summary>

| # | Correct Answer | Explanation |
|---|---------------|-------------|
| 1 | **B) The client** | Clients (your browser, phone, etc.) send requests. Servers respond with data. |
| 2 | **B) A building with thousands of servers** | Data centers are huge facilities where companies like Google, Amazon, and Microsoft keep their servers running 24/7. |
| 3 | **B) DNS → Request → Response → Render** | First find the IP (DNS), ask for the page (request), receive it (response), then display it (render). |
| 4 | **C) 200** | 200 OK means everything worked perfectly and the server is sending you the page! |
| 5 | **C) Page not found** | 404 means the server couldn't find the page you asked for. |
| 6 | **B) Secure** | HTTPS = HyperText Transfer Protocol Secure. The S means encrypted! |
| 7 | **C) Encrypting data** | HTTPS uses SSL/TLS to encrypt data, making it unreadable to anyone who intercepts it. |
| 8 | **C) HTML** | HTML (HyperText Markup Language) defines the structure — headings, paragraphs, images, links. |
| 9 | **B) CSS** | CSS (Cascading Style Sheets) handles visual styling — colors, fonts, spacing, layout. |
| 10 | **C) JavaScript** | JavaScript handles interactivity — button clicks, animations, dynamic content loading. |
| 11 | **A) Document Object Model** | The DOM is the browser's tree-like representation of an HTML document. |
| 12 | **C) Only on YOUR screen** | DevTools edits are local only. Refreshing resets everything. You can't change the real website! |

</details>
