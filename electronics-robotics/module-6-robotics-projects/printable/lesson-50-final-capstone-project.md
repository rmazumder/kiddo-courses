## Choose Your Challenge

![Four Capstone Choices](../images/lesson-50-four-capstone-choices.png)

**Pick ONE project to complete as your capstone:**

### Project 1: Firefighting Robot 🚒
**Detect a flame and extinguish it!**
- Add flame sensor (detects infrared from candle)
- When flame detected: stop, beep, turn fan on
- Challenge: Navigate maze while scanning for flame

### Project 2: Maze Runner 🌀
**Navigate a cardboard maze autonomously!**
- Use ultrasonic sensor to detect walls
- Scan and choose the clearest path
- Challenge: Complete the maze in fastest time

### Project 3: Robot Butler 🤖
**Greet people and follow them!**
- Use distance sensor to detect persons
- Move forward when person is near
- Use LEDs and buzzer to greet
- Challenge: Follow a moving target

### Project 4: Art Robot 🎨
**Draw patterns and designs!**
- Attach a marker to your robot arm
- Program geometric patterns (square, spiral, circle)
- Use pen-up/pen-down servo control
- Challenge: Create the most creative design

## The Engineering Design Process

![Design Build Test Cycle](../images/lesson-50-design-build-test-cycle.png)

**Every robot project follows the same cycle:**

```
           ┌─────────────────────────────┐
           │ 1. PLAN IT                  │
           │ Sketch, list sensors,       │
           │ write pseudocode            │
           └──────────┬──────────────────┘
                      │
           ┌──────────▼──────────────────┐
           │ 2. BUILD IT                 │
           │ Wire components,            │
           │ assemble chassis            │
           └──────────┬──────────────────┘
                      │
           ┌──────────▼──────────────────┐
           │ 3. CODE IT                  │
           │ Write Arduino program,      │
           │ test individual functions   │
           └──────────┬──────────────────┘
                      │
           ┌──────────▼──────────────────┐
           │ 4. TEST IT                  │
           │ Run full test, measure,     │
           │ record results              │
           └──────────┬──────────────────┘
                      │
           ┌──────────▼──────────────────┐
           │ 5. FIX IT                   │
           │ Debug, improve, optimize    │
           │ (if needed, back to PLAN)   │
           └──────────┬──────────────────┘
                      │
         (Repeat until success!)
```

---

## Project Checklist

### Planning Phase (Day 1)
- [ ] Choose your capstone project
- [ ] Sketch your robot design
- [ ] List all sensors and components needed
- [ ] Write pseudocode (English plan before real code)
- [ ] Draw wiring diagram

### Building Phase (Day 2)
- [ ] Gather all components
- [ ] Wire sensors and motors
- [ ] Test each component individually
- [ ] Integrate all components
- [ ] Power up and check for errors

### Coding Phase (Day 2-3)
- [ ] Start with movement functions
- [ ] Add sensor reading code
- [ ] Write decision logic (if-else)
- [ ] Test each function separately
- [ ] Integrate all functions

### Testing Phase (Day 3)
- [ ] Run full program
- [ ] Test in real environment
- [ ] Measure performance (time, accuracy, reliability)
- [ ] Record observations

### Optimization Phase (Day 4)
- [ ] Identify problems
- [ ] Improve code efficiency
- [ ] Adjust sensor values
- [ ] Re-test and compare
- [ ] Document final results

## Success Criteria

**Your robot project is successful when:**

✅ All sensors read correctly
✅ Motors respond to sensor input
✅ Code compiles without errors
✅ Robot completes the main task (flame detection, maze, etc.)
✅ You can explain how it works to someone else
✅ You've documented your process and improvements

## Quick Quiz

**Q1:** What are the 5 phases of the design process?
**A:** Plan, Build, Code, Test, Fix.

**Q2:** Why is testing important?
**A:** To verify your robot works as designed and identify problems to fix.

**Q3:** Can you return to earlier phases if testing fails?
**A:** YES! The design cycle is circular — go back and improve!
