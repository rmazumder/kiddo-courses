📚 Key Words — Learn These!
🔔 Hz
tone() Function
An Arduino command that makes the buzzer play a musical note. You give it a pin, a frequency (like 440 for A4), and how long to play.
💡 V
Voltage Drop
The voltage used up across a component. A red LED drops ~1.9V; a blue LED drops ~3.2V. Resistor + LED drops always add up to 5V!
🏃 ⎯
State Machine
A way of programming where the robot shows different lights and sounds for each state it is in: Forward, Backward, Turning, Obstacle!
📡 PWM
PWM Signal
A signal that switches rapidly between 0V and 5V. The buzzer uses PWM to make sound. Your multimeter reads the average voltage (~2.5V).
🔌 Robot LED Layout and Wiring
Front of Robot: [Green] [Blue] [Green] <-- Headlights | | | | (status) | | | +---+----------------+---+ | | | [ROBOT] | | | +---+----------------+---+ | | [Red] [Red] <-- Taillights Pin Assignments: Left Green LED -> Pin A1 + 330 ohm resistor Right Green LED -> Pin A2 + 330 ohm resistor Left Red LED -> Pin A3 + 330 ohm resistor Right Red LED -> Pin A4 + 330 ohm resistor Blue Status LED -> Pin 13 + 330 ohm resistor Buzzer (passive) -> Pin 8 (no resistor needed) Wiring each LED: Arduino Pin ---[330ohm]--- LED(+) ---LED(-) --- GND
Mount LEDs where they look like headlights and taillights. Use the small breadboard on the robot or tape wires along the chassis. Make it look cool — this is YOUR robot's face!
⚡ State-Based Light Patterns
Robot State | Green LEDs | Red LEDs | Blue LED ------------------+------------+-----------+---------- Driving FORWARD | BOTH ON | both off | ON Going BACKWARD | both off | BOTH ON | off Turning LEFT | LEFT on | LEFT on | off Turning RIGHT | RIGHT on | RIGHT on | off OBSTACLE SPOTTED | flashing | flashing | flashing STOPPED / waiting | all off | all off | ON Buzzer sounds: Obstacle detected -> two quick beeps (1000Hz, 1500Hz) Reversing -> classic BEEP BEEP (800Hz x2) Turning -> short chirp (2000Hz, 50ms) Happy (clear path)-> C5, E4, G4 melody
Think of it like a car: green lights = going forward, red lights = reversing, flashing = WARNING! The buzzer is your robot's voice — it tells everyone what the robot is thinking!

Upload this code and watch your robot wake up! LEDs cascade on, everything flashes, it plays a little melody. THAT is how a proper robot says hello!
✅ Build Steps — Did You Complete These?
  1. 1 Wire 2 green LEDs (pins A1, A2) each with a 330Ω resistor to GND
  2. 2 Wire 2 red LEDs (pins A3, A4) each with a 330Ω resistor to GND
  3. 3 Wire blue LED (pin 13) with a 330Ω resistor to GND
  4. 4 Wire passive buzzer: pin 8 (+) to GND (−), no resistor needed
  5. 5 Call setupLights() in setup(), then call startupSequence()
  6. 6 Add lightsForward(), soundObstacle() etc. to your obstacle-avoidance loop

🔧 Troubleshooting Quick Guide
ProblemQuick Fix
LED won't lightCheck polarity: long leg to resistor, short leg to GND
Buzzer very quietUse passive buzzer for tone(); active buzzer needs direct power
Wrong notes playingDouble-check frequency values in NOTE_ definitions
Robot won't moveMotor code may be missing — merge both sketches
LEDs flickerNormal! Motors cause small power fluctuations
Wand reads 0V on LEDMeasure ACROSS the LED, not the resistor
