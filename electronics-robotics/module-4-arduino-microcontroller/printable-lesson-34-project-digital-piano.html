📚 Key Words — Learn These!
🎵 Hz
Frequency (Hz)
How many times per second a sound wave vibrates. Higher Hz = higher note! Middle C = 262 Hz. Top C = 523 Hz. Double the Hz, double the pitch!
📈 [ ]
Array
A list of values stored together in one variable. Like a numbered mailbox row: notes[0] = C4, notes[1] = D4, and so on!
📀 I2C
I2C Protocol
A way to talk to the LCD using only 2 wires (SDA + SCL). The Arduino sends messages and the LCD listens. Saves lots of wiring pins!
🏗 PWM~
INPUT_PULLUP
A built-in Arduino trick that pulls button pins HIGH by default using internal resistors. Button pressed = LOW. No external resistors needed!
🎹 How the Digital Piano is Wired
DIGITAL PIANO CIRCUIT OVERVIEW ================================ [Arduino Uno] | |-- Pin 2 ---[BUTTON 1]--- GND (plays C4 = 262 Hz) |-- Pin 3 ---[BUTTON 2]--- GND (plays D4 = 294 Hz) |-- Pin 4 ---[BUTTON 3]--- GND (plays E4 = 330 Hz) |-- Pin 5 ---[BUTTON 4]--- GND (plays F4 = 349 Hz) |-- Pin 6 ---[BUTTON 5]--- GND (plays G4 = 392 Hz) |-- Pin 7 ---[BUTTON 6]--- GND (plays A4 = 440 Hz) |-- Pin 12 ---[BUTTON 7]--- GND (plays B4 = 494 Hz) |-- Pin 13 ---[BUTTON 8]--- GND (plays C5 = 523 Hz) | |-- Pin 8 ---[BUZZER+]---[BUZZER-]--- GND |-- Pin 9 ---[330ohm]---[LED Red]--- GND (C,D,E = low) |-- Pin 10 ---[330ohm]---[LED Green]-- GND (F,G = mid) |-- Pin 11 ---[330ohm]---[LED Blue]--- GND (A,B,C5= high) |-- A4 ---[LCD SDA] 5V ---[LCD VCC] |-- A5 ---[LCD SCL] GND ---[LCD GND]
Press a button → buzzer plays the note → LED lights up → LCD shows the note name and frequency!
💻 Key Code Patterns
// Arrays hold all 8 note frequencies and names int noteFreqs[] = {262, 294, 330, 349, 392, 440, 494, 523}; String noteNames[] = {"C4", "D4", "E4", "F4", "G4", "A4", "B4", "C5"}; // In loop(): scan every button with a for loop for (int i = 0; i < numButtons; i++) { if (digitalRead(buttonPins[i]) == LOW) { tone(buzzerPin, noteFreqs[i]); // play the note lcd.print(noteNames[i]); // show on screen } } if (!anyPressed) noTone(buzzerPin); // stop when released
The for loop scans all 8 buttons automatically — one loop does the job of 8 separate if statements!

PHASES AT A GLANCE ================== Phase 1 (5min) Plan the 8-note piano layout (+20 XP) Phase 2 (15min) Wire 8 buttons using INPUT_PULLUP (+40 XP) Phase 3 (5min) Wire piezo buzzer to pin 8 (+10 XP) Phase 4 (5min) Wire LCD via I2C (A4=SDA, A5=SCL) (+10 XP) Phase 5 (10min) Wire 3 indicator LEDs (pins 9,10,11) (+20 XP) Phase 6 (15min) Upload full piano code & play! (+60 XP) Phase 7 (5min) Debug with Multimeter Wand (+40 XP) Phase 8 (bonus) Extra challenges (+30 XP bonus)
  1. 1Set up buttons in a row across the breadboard center gap. Connect each button to its Arduino pin (2,3,4,5,6,7,12,13) and the other side to GND rail.
  2. 2Buzzer: pin 8 → buzzer (+) → buzzer (-) → GND. LEDs: pin 9,10,11 → 330Ω resistor → LED → GND.
  3. 3LCD: 5V → VCC, GND → GND, A4 → SDA, A5 → SCL. Include LiquidCrystal_I2C library in your sketch.
  4. 4Upload the piano sketch. Press each button — you should hear a note, see an LED glow, and see the note name + Hz on the LCD!
  5. 5Use the Multimeter Wand to measure: 5V pin, button pins when pressed/released, buzzer pin while playing, LED pins when active.
Test each button one at a time BEFORE uploading the full piano code. Fix wiring issues early — it is much easier!

🔬 4 Things You Built and Tested Today!
A
8-Button Input Grid
Wire all 8 buttons to pins 2–7, 12, 13 using INPUT_PULLUP.
Each button pin reads ~5V idle and ~0V when pressed — you can scan them all in one for loop!
B
Buzzer + Frequency Arrays
Use tone(pin, noteFreqs[i]) inside the for loop.
Double the frequency = double the pitch! C5 (523 Hz) sounds exactly one octave higher than C4 (262 Hz)!
C
LCD Note Display
Show noteNames[i] and noteFreqs[i] on the LCD screen.
The LCD updates instantly as you press different buttons — it is a real-time instrument display!
D
Wand Debugging Session
Measure button pins, buzzer pin, LED pins, and 5V rail with the multimeter.
Buzzer pin shows ~2.5V average while playing — that is PWM! Your wand can see the tone happening!

ProblemQuick Fix

Button not registeringMeasure the pin — should read ~5V idle
LCD blank / no textRun I2C scanner sketch to find the LCD address
Wrong notes playingCheck buttonPins[] array order matches wiring
LED stays on alwaysCheck the noTone / LED-off block in the code
